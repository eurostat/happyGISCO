tools module¶

Library of simple tools for simple geographical data (geolocations and geocoordinates) handling and processin.

About

credits: gjacopo

version: 1 – since: Sat Apr 14 20:23:34 2018

Description

Perform various types of geographical transformations and ‘conversions’ in order to represent equivalently and uniquely geographical locations/toponames.

Dependencies

require: os, sys, math, np

optional: osgeo

call: settings

ibid, minor semi-axis equal to EARTH_RADIUS_POLAR.

class happygisco.tools.GeoCoordinate(*args, **kwargs)[source]¶

Bases: happygisco.tools._GeoLocation

Class of geographic/location attributes and methods used to define, describe and represent the geospatial status of an object.

The term _”location”_ in geography is used to identify a point or an area on the Earth’s surface or elsewhere.

This class emulates _GeoLocation. Inherit for instance methods _check_bounds() from original class to check for (lat, Long) coordinates consistency; instead, methods distance_to() (computation of great circle distance between geolocations) and bounding_locations() (computation of the bounding coordinates of all points) are overriden.

MIN_LATITUDE¶

dummy min latitude value in degree: -90.

MAX_LATITUDE¶

ibid for max latitude: 90.

MIN_LONGITUDE¶

dummy min longitude value in degree: -180.

MAX_LONGITUDE¶

ibid for max longitude: 180.

Initialization overriding _GeoLocation super initialization method.

>>> x = GeoCoordinate(*args, **kwargs)
classmethod bbox2latlon(bbox)[source]¶

Convert an AOI bounding box into the corresponding (lat, Lon, rad) format.

>>> lat, Lon, rad = GeoCoordinate.bbox2latlon(bbox)
Parameters:bbox (list) – a bounding box represented as a 4-lenght list with the (lat,Lon) coordinates of the South-West and North-East corners of the input polygon.
Returns:lat,Lon,rad (float) – parameters defining the CIRCUMcircle of the input bounding box bbox.

Examples

>>> bbox = [2.216707433489782, 48.72843804413901, 2.477292566510218, 48.98924195586099]
>>> lLr = GeoCoordinate.bbox2latlon(bbox)
>>> print(lLr)
    (2.347, 48.85884, 14.50401801879798)

As mentioned, no idempotence, but the centre of the bounding box is still preserved:

>>> GeoCoordinate.latlon2bbox(*lLr) == bbox
    False
>>> lLr_ = GeoCoordinate.bbox2latlon(GeoCoordinate.latlon2bbox(*lLr))
>>> print(lLr_)
    (2.347, 48.85884, 29.007998346748554)
>>> lLr_[:2] == lLr[:2] # are the coordinates of the centre preserved?
    True

Note

This method and latlon2bbox() are not idempotent (say it otherwise GeoCoordinate.latlon2bbox(GeoCoordinate.bbox2latlon(bbox) does not return bbox): see comments on CIRCUMcircle and INcircle; however, the centre of the bounding box box (hence the tuple (lat,Lon)) is preserved.

See also

latlon2bbox(), centroid(), distance_to_from()

classmethod distance(*args, **kwargs)[source]¶

Class method used for computing pairwise distances between given locations, passed as geographic coordinates.

>>> D = GeoCoordinate.distance(*args, **kwargs)
Parameters:

args (tuple) – a pair of locations represented as a tuple of (lat,Lon) coordinates.
Keyword Arguments:
 
  • dist (str) – name of the geo-principle used to estimate the distance: it is any string in ['great_circle','vincenty'] since they represente the Great Circle distance and the Vincenty distance; see geopy.distance() method; default to 'great_circle'.
  • unit (str) – name of the unit used to return the result: any string from the list ['m','km','mi','ft']; default to ‘km’.
Returns:

D (np.array) – matrix of pairwise distances computed in unit unit.
Raises:

happyError: – * when unexpected variable for lat/long; * when wrong unit/code for geodesic distance.

Examples

>>> GeoCoordinate.distance((26.062951, -80.238853), (26.060484,-80.207268),
...                        dist='vincenty', unit='m')
    3172.3596179302895
>>> GeoCoordinate.distance((26.062951, -80.238853), (26.060484,-80.207268),
...                        dist='great_circle', unit='km')
    3.167782321855102

See also

distance_to(), distance_to_from()
